Элементы меню

Тест

Открывает тестовое окно. Это простое окно имеет редактируемые текстовые области, в которые вы можете поместить html код. Нажав кнопку Тест вы увидите, что произойдет при работе фильтра. Вы можете держать это окно открытым пока пока редактируете фильтр.

Кодирование / декодирование Base64

Кодирует содержимое буфера. Если вы поместили какой-либо простой текст в буфер, то использовав Кодировать в Base64, вы закодируете текст в формат Base64. Вы можете извлечь текст нажав Ctrl+V. Если вы поместили в буфер текст кодированный в Base64, то использовав пункт меню Декодировать из Base64 вы можете декодировать его.

Элементы окна

Заголовок

Это - название фильтра которое будет отображаться в конфигурации или в окне отчета. Название является обязательным.

Автор

Имя автора фильтра. Пожалуйста не изменяйте имя у фильтра написанного не вами.

Версия

Вы можете использовать это опциональное поле для указания версии фильтра.

Комментарий

Комментарий объясняет, что фильтр делает. Комментарий виден в списке конфигурации, чтобы вы помнили назначение фильтра. Не пренебрегайте комментарием.

Приоритет

Это произвольное число которое определяет порядок в котором активные фильтры выполняются. Фильтры с более высоким приоритетом выполняются раньше (например фильтр с приоритетом 200 выполнится перед фильтром с приоритетом 100). Этот порядок очень важен: например, если вы используйте фильтр, который удаляет некоторые JavaScripts и фильтр, который включается JavaScript, первый должен работать раньше, или он отменит эффект последнего. По умолчанию приоритеты определены так, чтобы не происходило помех ( 1000 до 0). Когда вы пишете новый фильтр, вы должны выбрать приоритет для вашего фильтра. Рекомендуется использовать приоритеты свыше 500 для фильтров, которые удаляют части html кода и ниже 500 для фильтров, которые включают html код. Примечания: 1/ вы можете использовать отрицательные числа; 2/ фильтры могут иметь одинаковый приоритет, если их порядок не имеет значения; 3/ фильтры для исходящих, фильтры для входящих и текстовые фильтры - три независимых группы.

Предельный размер

Это - размер текста, который фильтр читает, когда пытается найти соответствие в данной позиции. Этот размер необходимо делать как можно меньше, но достаточным для срабатывания фильтра. Например, если ваш шаблон border=$AV(\1) размера 20 должно быть достаточно (граничные атрибуты и значение не превышает 20 символов).

Разрешить многократное использование

Обычно, когда фильтр находит соответствие, замещающий текст посылается браузеру, а сканирование продолжается до следующего совпадения. Ни этот фильтр, ни другие фильтры не сканирует замещающий текст. Но, если вы включите эту опцию, замещающий текст помещается в буфер памяти и сканируется следующим фильтром. Шаблон фильтра должен отличаться от замещающего текста кроме первого символа, т.к. происходит смещение на один символ вправо. Используйте эту опцию только когда это действительно необходимо, например, когда вы имеете несколько фильтров, которые обрабатывают один и тот-же тег.

Тип фильтра

Это поле сообщает к какому типу принадлежит данный фильтр. Он может быть фильтром исходящих заголовков (посланных вашим браузером сайту), фильтром входящих заголовков (посланных сайтом в ответ) или фильтром текста (html кода), который загружает ваш браузер. В случае фильтров заголовка, имя заголовка должно присутствовать в соответствующем поле (смотри ниже). В случае фильтров текста, должен присутствовать предельный размер.

Заголовок

Это - имя заголовка который фильтр будет обрабатывать. Он нечувствителен к регистру. Шаблоны или коды - не разрешены(т.е. без *, ?). Для фильтров исходящих заголовка, если область начинается с "URL" (независимо от того, что следует далее), фильтр прочитает запрошенный URL (включая ://) вместо данного заголовка. Используйте эту хитрость, чтобы тестировать URL и возможно изменять его (используется подобно прозрачной переадресации) или вызова $ команд

URL шаблон

Если это поле содержащее шаблон и URL (без ://) не соответствуют, фильтр будет обойден, то есть ничего не будет искаться в этом заголовке/тексте. Используйте эту область, если вы хотите, чтобы фильтр работал только для определенных URL. Общие шаблоны: $LST(список), (^$LST(список)) или $TYPE(htm).

Границы шаблона

Если эта область содержит шаблон, процесс обработки будет состоять из двух проходов: 1- поиск границ шаблона, 2- если они найдены поиск соответствий с шаблоном. Это похоже на то - как будто бы вы использовали шаблон (bounds)&&(match). Просто это делает определенные вещи более понятными. Используйте границы шаблона, когда возможно, т.к. в этом случае обработка идет быстрее, и иногда лучше! Например, чтобы найти <img> тег, содержащий ".gif", вы могли бы использовать границы: <img\s*> и шаблон: *.gif*. Если вы просто используете шаблон <img\s*.gif*>, то он должен устанавить полстраницы в курсив, если страница содержит
<i><img src="bullet.jpg">какое-либо .gif анимированное изображение</i>.

Совпадение шаблона

Это - шаблон для обнаружения, который будет замещен в пределах документа. Для фильтра текста, этот шаблон будет искаться во всех позициях html кода. Для фильтра заголовков, он - запустится в начале заголовка (например вы должны использовать *nasty, а не просто nasty, чтобы найти значение заголовка хоста www.nasty.com).

Замещающий текст

Эта область может содержать простой текст, замещающий код и замена-совместимые команды. Замещающий код вычисляется и выполняются команды для того, чтобы создать "расширенный текст". Для фильтров текста, каждое совпадение заменяется этим расширенным текстом. Для фильтров заголовков, целое значение заголовка заменяется расширенным текстом.
