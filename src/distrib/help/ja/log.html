Buttons

ボタン

Start / Stop

開始 / 停止

When you click on Start, events generated by Proximodo will be shown in the log window (according to options). If you click on stop, events are not caught by the log window anymore (they will be lost) and you will be able to read the content peacefully. When you close the window, it is turned off automatically.

「開始」をクリックすると、Proximodo で発生したイベント（オプションによります） がログウィンドウに表示されます。

「停止」をクリックすると、 表示が止まり、内容をゆっくり読むことが出来ます （その間のイベントを後から表示させることは出来ません）。 ログウィンドウを閉じている間は、自動的に停止状態になります。

Clear

消去

Click here whenever you want to empty the window.

クリックするとログが消去されます。

Options

オプション

View HTTP Events

HTTP イベント

If this box is on, the window will display events generated by the HTTP monitoring engine. See the next two options.

チェックを入れると、HTTP 監視エンジンで発生したイベントが表示されます。 以下の二つのオプションも参照して下さい。

View POST Data

POST データ

With this option on, HTTP events will include events of type "Received POST data from browser". The POST data content is entirely displayed along with the event description.

チェックを入れると、HTTP イベントとして、ブラウザから受け取った POST データも表示されます。 POST データはイベントの詳細も合わせ完全に表示されます。

Browser-side dialog

ブラウザ対話モード

When this option is on, HTTP events will include events of type "Received request from browser" and "Sent response to browser" (browser-side communications). If it is off, HTTP events will include events of type "Sent request to website" and "Received response from website" (internet-side communications).

チェックを入れると、HTTP イベントの種類が 「ブラウザからのリクエストを受信」と「ブラウザへレスポンスを送信」 （Proximodo とブラウザの間の通信を表示）になります。

チェックを外すと、「ウェブサイトへリクエストを送信」 と「ウェブサイトからのレスポンスを受信」（Proximodo とインターネットの間の通信を表示）になります。

View Proxy Events

Proxy イベント

Proxy events will be displayed. They are "Proxy started/stopped", "Connection accepted/refused", "New request started/finished", "New socket opened/closed".

チェックを入れると、 以下のプロクシイベントが表示されます： 「プロクシ接続を開始しました／停止しました」 「接続しました／拒否されました」 「新しいリクエストを受信しました／完了しました」 「新しいソケットを作成しました／閉じました」。

View Filter Events

フィルタイベント

Active filters that match will generate two lines (one to show what matched and one to show the replacement text).

チェックを入れると、有効なフィルタがマッチ した時に二行（マッチした内容と、置換テキスト）で表示されます。

View $LOG() Events

$LOG() イベント

Events generated by $LOG commands in filters will be displayed.

チェックを入れると、フィルタ内で使用された $LOG の結果が表示されます。
